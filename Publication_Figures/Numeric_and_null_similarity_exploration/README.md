# Numeric and Null Similarity Exploration

## Status

This folder contains exploratory analysis outputs only.

These graphs were **not used in the final publication**. They are useful for internal inspection, but the results are imperfect because they rely on heuristic parsing of complex free-text responses generated by an LLM and free-text human review fields.

## What Data This Uses

The script in this folder compares:

- Copilot processed dataset entries:
  - `Copilot_Processed_Datasets_JSON/Copilot_222_v2_Processed_2026-03-02_Updated_Metadata/*.json`
- Human DOME registry review entries:
  - `DOME_Registry_Human_Reviews_258_20260205.json`

Matching is done by normalized DOI (lowercased, trimmed).

Only non-`publication/*` fields are compared.

## Why This Is Exploratory (Important Nuances)

This analysis is intentionally simplified and has known limitations:

- Both sides contain free text with inconsistent phrasing and formatting.
- Numeric and percentage extraction uses regex, which can miss context or capture irrelevant numbers.
- Yes/No detection is keyword-based and can be ambiguous in mixed or multi-part answers.
- "Not enough information" handling uses pattern rules and partial scoring, not semantic understanding.
- URL comparison checks normalized URL token overlap, but does not fully model citation intent or equivalence.
- Some fields contain multi-line sub-answers where colon-separated parsing is heuristic.

Because of these factors, this output should be read as directional/exploratory, not definitive benchmarking.

## What Each Graph Represents

1. `graph_1_numeric_percent_direct_match_by_field.png`
- Direct numeric/percentage match rate by field for rows where both sides had numeric content.

2. `graph_1b_numeric_percent_breakdown_by_field.png`
- Numeric/percentage breakdown by field (match, mismatch, one-sided numeric availability).

3. `graph_2_not_enough_info_alignment_by_field.png`
- Alignment of Copilot "not enough information" outputs against strong-negative/null-like human responses.

4. `graph_3_yesno_alignment_by_field.png`
- Yes/No alignment by field where both sides were classifiable.

5. `graph_3b_yesno_breakdown_by_field.png`
- Yes/No detailed breakdown by field (full/partial/mismatch/one-sided availability).

6. `graph_4_url_alignment_by_field.png`
- URL overlap alignment by field where both sides contained at least one URL.

7. `graph_4b_url_breakdown_by_field.png`
- URL detailed breakdown by field (full/partial/no overlap/one-sided URL availability).
